# 字幕系统

ClipperStudio 的字幕功能分两块：**生成字幕**和**编辑字幕**。生成靠 AI 自动识别，编辑靠你自己调整。

## 生成字幕（ASR 语音识别）

### 什么是 ASR？

ASR 就是「自动语音识别」，简单说就是让电脑听视频里说的话，然后自动变成文字字幕。

ClipperStudio 把 ASR 服务作为本地后台进程运行。打开侧边栏「语音识别」页，你会看到左边是「**语音识别设置**」和「**本地服务设置**」两个 Tab，右边是「**ASR 任务**」和「**本地 ASR 日志**」。

### 本地服务的两种部署方式

「本地服务设置」里可以选择 ASR 服务的启动方式：

| 模式 | 说明 |
|------|------|
| **Docker 方式** | 用 Docker 容器跑 ASR 服务，环境干净，省心。需要电脑上装好 Docker。 |
| **本地源码模式** | 直接调用本地 Python 环境跑 ASR。适合不想装 Docker 的用户。 |

**本地源码模式**第一次使用前要先准备 Python 环境：

- macOS / Linux — 点「打开终端运行 setup.sh」按钮，按提示走完即可
- Windows — 在 ASR 仓库目录下手动运行 `setup.bat`

ClipperStudio 会自动检测虚拟环境（venv）或便携版 Python 是否就绪，没就绪时会在界面上告诉你下一步要做什么。

### 启动与停止服务

无论你用 Docker 还是本地源码，启动 / 停止都在同一个面板：

- 状态指示灯：绿色「运行中」、黄色「启动中…」、红色「错误」、灰色「已停止」
- 一键「**启动服务**」/「**停止服务**」
- 第一次启动如果检测到模型还在下载，会提示「模型正在下载中，首次启动可能较慢…」，耐心等就行
- 用 Docker 时遇到容器名冲突，ClipperStudio 会引导你强制清理并重建

### 实时日志

右侧的「本地 ASR 日志」面板会滚动显示最近 2000 行日志，启动慢、识别报错时直接看这里就能定位问题。

### 局域网访问

「语音识别设置」里有一个「**允许局域网访问**」开关：

- **关闭**（默认）— 服务只监听 `127.0.0.1`，外部设备连不进来，最安全
- **开启** — 监听所有网卡（`0.0.0.0`），同一局域网里的其他设备可以通过你的 IP 访问这个 ASR 服务，状态会标注「已开放」

只在自己电脑上用 ClipperStudio 的话，保持默认就行。

### 开始识别

1. 确保你已经在设置里配好了 ASR 引擎
2. 在视频列表里选一个视频
3. 点「生成字幕」按钮
4. 等任务跑完，字幕就会出现在视频下方

识别的速度取决于视频长度和你用的引擎。一段 3 小时的直播，用本地 ASR 可能要跑一会儿，用云端服务就快很多。

### 识别精度怎么样？

本地模型（比如 Qwen3 ASR）对中文直播场景识别效果还不错，偶尔会有同音字错误。如果你对精度有要求，可以：

1. 生成后手动编辑修正（推荐）
2. 换一个精度更高的引擎

## 查看字幕

生成好的字幕会显示在视频下方的字幕列表里。每一行字幕显示：
- 开始和结束时间
- 识别出来的文字内容

点击任意一行，播放器会自动跳到那个时间点播放。

## 编辑字幕

在工作台里切换到「编辑模式」，就能编辑字幕了。

### 基本操作

- **修改文字** — 直接点那行字幕的文字区域就能编辑
- **调整时间** — 拖动字幕段的时间边界，或者手动输入时间码
- **合并字幕** — 选中两行相邻的字幕，合并成一行
- **拆分字幕** — 把一行太长的字幕拆成两行
- **快捷打点** — 字幕行右侧的 `[` / `]` 按钮可以把当前行的开始 / 结束时间直接喂给切片选区

::: tip
字幕列表用的是虚拟滚动，几千条也不会卡。播放过程中当前字幕会自动跟随高亮，不用手动找。
:::

### 搜索字幕

ClipperStudio 支持全文搜索。字幕面板上方有个搜索框，输入关键词就能搜到所有包含这个词的字幕行。搜索做了防抖处理（输完一会儿才会触发），打字时不会卡顿。

这个功能很实用——比如你想找主播说过某句话的时刻，搜一下就能定位到。

## 导出字幕

编辑好之后，可以把字幕导出为文件。支持的格式：

| 格式 | 说明 |
|------|------|
| SRT | 通用字幕格式，大部分播放器和平台都支持 |
| ASS | 高级字幕格式，支持样式和特效 |
| VTT | Web 字幕格式，主要用于网页播放器 |

在字幕面板里选「导出」，选好格式和保存位置就行。

::: warning
字幕使用的是绝对时间（基于直播的实际时间），所以导出时 ClipperStudio 会自动转换为文件内的相对时间。你不需要操心这个转换，直接导出就能用。
:::

## 常见问题

**识别出来的字幕时间对不上怎么办？**

可以手动调整时间。在编辑模式下拖动字幕的时间边界就行。如果整体偏移，可能是直播有延迟导致的，调整一下就行。

**能手动创建字幕吗？**

可以。在编辑模式下，你可以在任意位置手动添加字幕行，不用非得用 ASR 生成。

**支持多语言吗？**

取决于你用的 ASR 引擎。有些引擎支持多语言识别，具体看引擎的文档。
