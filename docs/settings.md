# 设置与依赖管理

ClipperStudio 的设置页面用来管理应用本身的行为和外部依赖。

## 依赖管理器

这是设置里最重要的一个功能。ClipperStudio 做切片、压弹幕这些事情，背后需要一些外部工具（主要是 FFmpeg 和 DanmakuFactory）。依赖管理器就是帮你管好这些工具的。

### 怎么用

打开设置页面，你会看到所有依赖在同一份列表里，每一项都标着是核心依赖还是可选依赖，以及它当前的状态：

| 状态 | 含义 |
|------|------|
| 已安装（依赖管理器） | ClipperStudio 帮你装的，可以「打开目录」看一眼或者「卸载」 |
| 系统已安装 | 在你的 PATH 里检测到了，直接复用，不重复装 |
| 未安装 | 还没准备好，点「安装」即可 |
| 下载中 / 安装中 / 错误 | 当前操作的中间状态 |

如果某个依赖显示「未安装」，点「**安装**」按钮就行。ClipperStudio 会自动下载对应平台的版本，不需要你自己去找安装包。

::: tip
FFmpeg 是核心依赖，建议装好软件后第一件事就把它装上。没有 FFmpeg 的话切片功能用不了。
:::

### 安装进度与取消

安装时会显示一条三段式进度：「**下载中 → 解压中 → 验证中**」，每个阶段都能看到具体百分比。

下载阶段右侧有「**取消安装**」按钮，网络不好或下错版本时可以随时停掉。如果下载明显卡住，旁边还会出现「下载缓慢？尝试手动下载」的链接，点开就是直链地址，下载完用「自定义路径」喂给 ClipperStudio 即可。

### 下载代理

国内网络访问 GitHub 偶尔会抽风。依赖管理面板里有一个「**下载代理**」输入框，填上你的 HTTP 代理地址再保存，之后所有依赖下载都会走代理。

### 手动指定路径

如果你电脑上已经装过 FFmpeg 或者 DanmakuFactory，不需要重复安装。在依赖项的操作里选「**自定义路径**」，选到已有的可执行文件就行。

ClipperStudio 启动时也会自动扫描系统 PATH，发现已经装好的工具会直接识别到。

### 重新检测

如果装了新的依赖但 ClipperStudio 没检测到，点「重新检测」按钮就行。

### 卡住的下载会自愈

如果上一次下载意外中断（比如应用被强退），下次启动时这条任务会被自动重置回「未安装」状态，不会卡在中间状态让你手足无措。

## 应用设置

### 通用设置

- **语言** — 界面语言（目前支持中文和英文）
- **主题** — 深色 / 浅色 / 跟随系统
- **启动时行为** — 是否最小化到托盘、是否记住上次的工作区

### 切片设置

- **默认输出目录** — 切片文件保存到哪里
- **默认编码预设** — 快速 / 标准 / 高质量
- **文件名模板** — 切片文件怎么命名，支持变量替换

### 播放器设置

- **弹幕透明度** — 弹幕叠加层的透明度
- **弹幕密度** — 降采样级别
- **自动播放** — 打开视频时是否自动播放

## 配置文件在哪？

ClipperStudio 的配置文件存在用户数据目录下：

- **Windows** — `%APPDATA%/clipper-studio/`
- **macOS** — `~/Library/Application Support/clipper-studio/`
- **Linux** — `~/.config/clipper-studio/`

一般情况下你不需要手动编辑这些文件，通过设置页面操作就够了。但如果你想把配置备份一下或者迁移到另一台电脑，直接复制这个目录就行。
