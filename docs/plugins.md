# 插件系统

ClipperStudio 的一些功能是通过插件实现的，比如 ASR 引擎、弹幕源、导出格式等。这样做的好处是你能按需安装，不需要的功能不装也不影响使用。

## 什么是插件？

插件就是给 ClipperStudio 添加额外能力的小程序。它们独立于主程序运行，ClipperStudio 通过标准化的接口和它们通信。

你不需要理解插件的技术细节，只需要知道：**想加什么功能，去插件市场装一个就行。**

## 插件类型

ClipperStudio 的插件分三类：

| 类型 | 说明 | 例子 |
|------|------|------|
| 服务级 | 后台常驻运行的服务程序 | ASR 引擎、录制工具 |
| 工具级 | 用完就退的一次性工具 | 弹幕转换、字幕导出 |
| 适配器 | 帮 ClipperStudio 识别特定格式 | 录播姬目录适配器 |

## 管理插件

在设置页面能找到「插件管理」入口。这里你可以：

- **查看已安装的插件** — 每个插件的名字、版本、运行状态
- **安装新插件** — 从插件列表里选择安装
- **更新插件** — 有新版本时提示你更新
- **卸载插件** — 不需要了就卸掉，不影响其他功能
- **启动 / 停止** — 服务级插件可以手动启停

## 常用插件介绍

### ASR 引擎（语音识别）

ASR 引擎用来把视频里的语音转成字幕。ClipperStudio 把它做成了独立的本地服务（基于 Qwen3 ASR），可以选择两种部署方式：

- **Docker 方式** — 用容器跑，环境干净，需要预装 Docker
- **本地源码模式** — 直接用本地 Python 跑，省掉 Docker 这层依赖

详细的启动、停止、日志查看、局域网访问开关，请看 [字幕系统](/subtitle) 里的「生成字幕（ASR 语音识别）」一节。

### 弹幕转换工具

DanmakuFactory — 把 B 站的 XML 弹幕转成 ASS 字幕格式，弹幕压制功能需要它。在依赖管理里可以一键安装。

### 工作空间适配器

让 ClipperStudio 能识别不同录制工具的目录结构。目前内置了录播姬（BililiveRecorder）的适配器。如果你用的是其他录制工具，可以看看有没有对应的适配器插件。

## 安装插件

一般有两种方式：

1. **通过依赖管理器** — 在设置页面，ClipperStudio 会自动检测缺少的依赖，点一下就能安装。这是最简单的方式。

2. **手动安装** — 下载插件文件，放到插件目录下，然后在插件管理页面刷新。

大部分情况下你只需要用第一种方式就够了。

## 注意事项

- 插件和主程序之间有版本兼容性检查，版本不匹配会提醒你
- 卸载插件前记得确认没有正在进行的任务依赖它
- 服务级插件（比如 ASR）安装后需要手动启动一次
